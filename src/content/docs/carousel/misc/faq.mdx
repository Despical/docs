## Which Java versions are supported?

The plugin is compiled using **Java Development Kit (JDK) 21**. Therefore, **Java 21 or newer** is strictly required to run **Carousel**.

<Aside type="note">
    We adopt the latest stable Java versions to ensure the best performance and security. If you are using an older version of Java, the plugin will fail to load.
</Aside>

---

## Which Minecraft versions are supported?

**Carousel** supports a wide range of versions to ensure maximum compatibility for your server:

* **Supported Versions:** Full and native support for **1.8 and all newer versions** (1.21+).

<Aside type="caution">
    Sub-versions like 1.20.4 or 1.21.8 are not officially supported. Please ensure you are on the final minor release of your chosen major version.
</Aside>

## Why do my entities disappear randomly?

The plugin uses a unique, invisible identifier (tag) stored in the entity's custom name to track which horses belong to a carousel. This allows the plugin to clean up "ghost" entities that might get stuck after a restart.

**The internal tag used is:** `§8§8§0§c§d`

<Aside type="caution" title="Avoid using this tag">
    If you manually give any entity a custom name containing this exact color code sequence (`§8§8§0§c§d`), the plugin will mistake it for a stuck carousel entity.

    As a result, that entity will be **deleted automatically** when the server restarts or when you run the `/carousel purge` command.
</Aside>

---

## Entities stuck after a reload or crash?

If your server crashes, shuts down unexpectedly, or if you reload while a carousel is spinning, you might notice "ghost" horses remaining in the world.

<Aside type="tip" title="Resolved in v1.8.9+">
    This issue has been automatically resolved starting from version **1.8.9**. The plugin now intelligently detects and cleans up these stuck entities when the server starts up.
</Aside>

However, if you still encounter stuck entities or want to manually clean them up without restarting, we have added a specific command for this:

```bash
/carousel purge
```
Running this command will instantly scan the worlds and remove any stuck carousel entities.
